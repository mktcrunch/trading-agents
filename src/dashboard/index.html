Two Traders, Specialist Crews, Full Traceability

Twin Ledger is a live head-to-head trading experiment on Google Cloud. Two competing traders — Baseline and Internal — each start with $100,000 on a dedicated Alpaca paper account. Every trader is a Vertex AI Agent Engine deployment: a chat-mode coordinator routes work to task-mode specialist agents (data, signal) and deterministic execution tools (risk, orders, chase). Cloud Scheduler hits each engine directly via streamQuery; scheduled messages bypass LLM chaining and run the deterministic pipeline. Every action is logged to GCS with a trace_id and surfaced in this dashboard — equity charts come from Alpaca portfolio history.

Risk & exit mechanics (intraday)

Baseline risk profile

  • Base stop: fixed −1% return from entry
  • Trailing: pure LLM planner (activation 0.5–4%, lock 45–80%)
  • Prediction gate: none — stops execute immediately
  • EOD exit: losers below −0.95% in final ~6 min before close
  • State: persisted to GCS risk_state_baseline.json

Internal risk profile

  • Base stop: ATR × 1.5 dynamic (fallback −1%)
  • Trailing: hybrid — 1% activation / 70% lock floor + LLM tightening
  • Prediction gate: 15-min internal signal must agree before stop sell
  • EOD exit: same −0.95% / ~6 min window as Baseline
  • Held exits: logged as risk_held when model disagrees

Post-open chase rules

  • Scan overnight DAY/OPG limits placed since prior close
  • Skip if >70% filled or already chased
  • 30-min volatility gate (std dev + range) must be calm
  • Cancel remainder → market order; audit skip reason if gated
What each competing trader does differently

Baseline · System A

Technicals + pure LLM reasoning · shared multi-agent crew
  • Agents: Coordinator orchestrates data → signal → risk → execution
  • Signal: RSI, MACD, Bollinger, news — public data only
  • Sizing: LLM size_pct → equal-weight allocation (max 10%)
  • Overnight: DAY limit orders, chased if unfilled
  • Trailing stops: Pure LLM-planned activation & profit lock
  • Base stop: Fixed −1% return threshold
  • Prediction gate: None — exits execute immediately
  • Edge hypothesis: Structured LLM reasoning beats gut feel

Internal · System B

Internal signal + strategy + data sources · same crew pattern
  • Agents: Same coordinator/data/signal/risk pipeline with richer data tools
  • Signal: Internal signal layer + proprietary data sources + technicals
  • Sizing: Internal strategy from signal snapshot (LLM size as context)
  • Overnight: Same execution path, signal snapshot passed to allocator
  • Trailing stops: Scripted 1%/70% floor + LLM refinement (hybrid)
  • Base stop: ATR × 1.5 dynamic stop (fallback −1%)
  • Prediction gate: 15-min internal signal must agree before stop exit
  • Edge hypothesis: Proprietary signal + data stack is a real alpha source
